# Checks — 이 기능의 완성 기준

## Stage 0 완료 기준

- [ ] 레포 구조 요약이 작성됨
- [ ] 웹 UI 구현 후보 위치가 확인됨
- [ ] API/job 구현 후보 위치가 확인됨
- [ ] 기존 Conclave engine 재사용 포인트가 확인됨
- [ ] 다음 구현 단계가 작게 나뉘어 있음
- [ ] 불확실한 점이 `결정 필요`로 표시됨

## Stage 1 완료 기준 — 일반 유저용 언어/정보 구조

- [ ] 사용자용 용어 매핑이 코드 또는 config로 정의됨
- [ ] `통과`, `안 맞음`, `확인 부족`, `결정 필요`, `위험 감수` 상태가 정의됨
- [ ] UI에서 직접 노출할 label과 내부 enum이 분리됨
- [ ] 기존 영어/개발자 용어가 주요 화면에 직접 노출되지 않음

## Stage 2 완료 기준 — 아이디어 입력 + 제품 설명서 초안

- [ ] 사용자가 raw idea를 입력할 수 있음
- [ ] Conclave가 이해한 내용을 보여줄 수 있음
- [ ] 사용자가 이해 내용이 맞는지 수정/확인할 수 있음
- [ ] 제품 설명서 초안을 생성하거나 저장할 수 있음

## Stage 3 완료 기준 — 맞춤 질문 생성

- [ ] 아이디어에 맞는 질문 후보를 생성하는 인터페이스가 있음
- [ ] 질문에 추천 답변과 이유가 포함됨
- [ ] 빠르게/정확하게/꼼꼼하게 모드에 따라 질문 수가 제한됨
- [ ] 질문 답변이 제품 설명서에 반영됨
- [ ] LLM이 없을 때도 mock/fallback으로 UI 개발이 가능함

## Stage 4 완료 기준 — 꼭 들어가야 할 항목 카드

- [ ] 제품 설명서에서 항목 카드를 만들 수 있음
- [ ] 각 항목은 category, priority, status를 가짐
- [ ] 각 항목은 완성 기준 또는 확인 방법을 가질 수 있음
- [ ] 항목 상태가 사용자용 label로 표시됨

## Stage 5 완료 기준 — 확인 결과 + 고쳐보기 loop

- [ ] 항목별 확인 결과를 보여줄 수 있음
- [ ] 안 맞음 항목에 확인 근거가 표시됨
- [ ] 확인 부족 항목에 이유가 표시됨
- [ ] `고쳐보기`, `작업으로 만들기`, `제품 설명서 수정`, `위험 감수` 행동이 있음
- [ ] fix loop attempt가 2회를 넘으면 사람 확인 필요로 전환됨

## Stage 6 완료 기준 — Builder Pack export

- [ ] 제품 설명서, 항목, 완성 기준, 확인 방법을 Claude Code용 문서로 export 가능
- [ ] export된 작업 문서는 현재 작업 범위만 포함함
- [ ] out-of-scope가 명확히 적혀 있음
- [ ] 테스트/확인 방법이 포함됨

## Stage 7 완료 기준 — 기존 Conclave job 연결

- [ ] 기존 review/autofix/audit job 중 재사용 가능한 경로를 연결함
- [ ] PR 또는 결과물의 상태가 항목 카드와 연결됨
- [ ] PR comment는 요약만 남기고 전체 결과는 앱으로 링크함
