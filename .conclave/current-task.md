# Current Task — Stage 0: 레포 탐색과 구현 계획만 작성

## 목표

Conclave 레포를 실제로 살펴보고, “일반 유저용 시각적 작업공간”을 어디에, 어떤 순서로 구현할지 계획한다.

## 절대 하지 말 것

- 코드 구현하지 말 것
- 대규모 리팩터링하지 말 것
- 새 dependency 설치하지 말 것
- DB migration 만들지 말 것
- 기존 CLI behavior 바꾸지 말 것
- 배포 관련 파일 수정하지 말 것

## 해야 할 일

1. 레포 구조를 파악한다.
2. 웹 앱 또는 대시보드가 어디에 있는지 찾는다.
3. 중앙 SaaS API 또는 Worker가 어디에 있는지 찾는다.
4. 기존 review/autofix/job 구조가 어디에 있는지 찾는다.
5. 기존 PRD/spec/audit 관련 코드가 어디에 있는지 찾는다.
6. 새 기능을 구현할 후보 위치를 정리한다.
7. 단계별 구현 계획을 `.conclave/docs/repo-findings.md`로 작성한다.

## 산출물

`.conclave/docs/repo-findings.md` 파일을 작성한다.

포함해야 할 내용:

```text
1. 현재 레포 구조 요약
2. 웹 UI 구현 후보 위치
3. API/job 구현 후보 위치
4. 기존 Conclave 엔진 재사용 포인트
5. 새로 만들어야 할 데이터 모델
6. Stage 1부터 Stage 7까지 구현 순서
7. 위험한 부분
8. 먼저 사용자에게 확인해야 할 결정
```

## 완료 조건

- 코드 수정 없이 탐색 결과 문서가 작성되어 있다.
- 다음 단계에서 어떤 파일을 수정할지 명확하다.
- 불확실한 점은 추측하지 않고 `결정 필요`로 적혀 있다.
