# Claude Code Prompt — 실패한 Stage 고쳐보기

이전 작업에서 실패 또는 확인 부족이 발생했다.

실패/확인 부족 내용:

```text
<여기에 실패 내용 붙여넣기>
```

해야 할 일:

1. 실패 내용을 원인별로 분류한다.
2. 제품 설명서/결정이 부족해서 생긴 문제인지, 구현이 틀려서 생긴 문제인지 구분한다.
3. 수정 계획을 작성한다.
4. 필요한 경우 사용자에게 결정 질문을 만든다.
5. 구현 수정이 필요한 경우 최소 변경으로 고친다.
6. 관련 확인을 다시 실행한다.
7. 결과를 `통과`, `안 맞음`, `확인 부족`, `결정 필요` 중 하나로 보고한다.

제한:

- 같은 항목 자동수정은 최대 2회까지만 시도한다.
- 2회 실패하면 사람 확인 필요로 표시한다.
- 범위를 넓혀서 다른 기능을 새로 구현하지 않는다.

보고 형식:

```text
문제 분류
- ...

수정한 것
- ...

다시 실행한 확인
- ...

결과
- 통과 / 안 맞음 / 확인 부족 / 결정 필요

다음 행동
- ...
```
