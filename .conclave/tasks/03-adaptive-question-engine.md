# Stage 3 — 맞춤 질문 생성 엔진

## 목표

사용자의 아이디어에 맞춰 중요한 빈칸을 찾고, 질문 후보를 생성하고, 우선순위가 높은 질문만 보여준다.

## 질문 생성 원칙

템플릿 질문만 쓰지 않는다. raw idea에서 다음을 감지한다.

```text
- 핵심 사용자 행동
- 외부 연동
- 민감 데이터
- 권한/소유권
- 실패 상황
- 자동 실행 여부
- MVP 범위
- 제외할 기능
```

## 질문 카드 필드

```text
질문
추천 답변
추천 이유
선택지
직접 입력
나중에 정하기
```

## 모드

```text
빠르게 초안 만들기: 3~5개
정확하게 만들기: 8~12개
출시용으로 꼼꼼하게: 20개 이상 가능하되 접어서 표시
```

## 구현 방식

- LLM adapter interface를 만든다.
- LLM unavailable 상태에서도 mock/fallback으로 개발 가능하게 한다.
- 질문 후보는 schema validation을 거친다.
- 사용자에게 처음부터 모든 질문을 보여주지 않는다.

## 완료 조건

- raw idea에서 question candidates 생성 가능
- 우선순위에 따라 표시 개수 제한 가능
- 추천 답변과 이유가 화면에 표시됨
- 답변이 제품 설명서 draft에 반영됨
