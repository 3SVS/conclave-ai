# Stage 5 — 확인 결과와 고쳐보기 2단계 루프

## 목표

각 항목의 확인 결과를 보여주고, 안 맞음/확인 부족 항목을 다시 고치는 흐름을 만든다.

## 결과 상태

```text
통과: 확인 근거 있음
안 맞음: 확인 근거 있는 실패
확인 부족: 증거 부족
결정 필요: 사용자 결정 필요
위험 감수: 사용자가 이번 버전에서 넘어감
```

## 2단계 loop

### 1단계 — 제품 설명서 고치기

결정이 부족해서 생긴 문제를 해결한다.

예:

```text
녹음 파일 보관 기간이 정해져 있지 않음
```

행동:

```text
사용자에게 질문 → 제품 설명서 수정 → 항목 업데이트
```

### 2단계 — 결과물 고치기

구현이 제품 설명서와 안 맞는 문제를 해결한다.

예:

```text
Linear 전송 전 확인 단계가 없음
```

행동:

```text
수정 계획 생성 → 사용자 승인 → builder prompt 또는 patch 생성 → 다시 확인
```

## 제한

- 자동수정은 항목당 최대 2회
- 2회 실패하면 사람 확인 필요
- 사용자가 승인하지 않은 patch를 자동 commit하지 않음

## 완료 조건

- CheckResult 표시 가능
- 실패 항목에서 FixLoop 생성 가능
- spec_fix와 implementation_fix 구분 가능
- attempt limit 동작
- 사용자에게 다음 행동이 명확히 표시됨
