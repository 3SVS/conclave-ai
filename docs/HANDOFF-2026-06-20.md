### 구현
- **central-plane** `src/workspace/experiment-outcome-scorecard.ts` — `computeOutcomeScorecard` 순수 헬퍼.

  (소유권 검증, candidate.benchmarkId로 벤치마크 로드, `{ok:true, scorecard}`)
- **dashboard** `src/lib/outcome-labels.mjs`(+`.d.mts`) 순수 매퍼 / `workspace-experiment-api.ts` `getOutcomeScorecard`+타입 /
  experiment 페이지 **Outcome quality** 섹션(grade 배지·메트릭 그리드·추천 다음 단계+이유·focus 칩·근거/수정/계획 버튼, openExp 변경 시 재로드) /

### Evolution action pack model
`apps/dashboard/src/lib/evolution-action-pack.mjs`(+`.d.mts`) 순수·결정론. 모든 노출 텍스트는 localized `s`(=`t.evolution`)로 주입(buildCandidatePrompt 컨벤션). `buildEvolutionActionPack(input, s)` / `buildEvolutionActionPackText(pack, s, meta)` / `resolveFocusItems` / `statusLabelFor`. 입력 타입(SavedExperiment/AgentBenchmarkResult/acceptanceItems)은 Stage 77 central-plane persistence 이전까지 안정.

### recommendedAction별 behavior
- accept: Decision·Evidence summary·Pre-merge checklist·Next review (코드 수정 지시 없음)
- fix_selected: Goal·Focus acceptance items·Constraints·Expected output·After completion (selected candidate 대상, "제품 의도 다시 쓰지 마라" 톤)
- rerun_experiment: Why rerun·Suggested setup·Candidate roles·How to compare (실험 자동생성 안 함)
- clarify_acceptance_items: Why clarify·Items needing clarification·Questions·After clarification (acceptance item 자동수정 안 함)
- create_benchmark: Why benchmark first·Required inputs·Steps·What to expect

### Focus item resolution
`suggestedFocusItemIds` → title는 basis candidate `benchmark.itemOutcomesByCandidate` → `remainingBlockers` → project acceptance items → **itemId fallback**(없는 title 발명 안 함). status 라벨: Issue found/Needs decision/Not verified.

### Dashboard UI
Outcome quality 섹션 내 **Evolution action pack** 서브카드 — Generate(benchmark snapshot fetch, best effort)·preview(action chip+target candidate+sections)·Copy(결정론 markdown). scorecard 로드 실패 시 needScorecard 안내. scorecard 있으면(create_benchmark/inconclusive 포함) 항상 생성 가능.

### Fix instructions와 구분 (rename 안 함)
Fix instructions=PR review/benchmark blocker로 한 구현 수정. Evolution action pack=outcome scorecard로 다음 루프(accept/fix/rerun/clarify/benchmark) 결정.

### i18n / tests
`evolution.*`(EN+KO+`.d.mts`, parity 10/10). `test/evolution-action-pack.test.mjs` 9개(액션별 pack, focus status, itemId fallback, acceptance-item title fallback, copy markdown+**userKey/token 미포함**). dashboard **150/150**·typecheck **54/54**·lint clean·build green.

### Known limitations
